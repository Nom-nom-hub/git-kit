# Contributing to Git-Kit

We welcome contributions to Git-Kit!

## Workflow
We use the **Git-Kit Methodology** for our own development.

1.  **Vision**: Check `CHARTER.md` (if it exists) or README for project goals.
2.  **Plan**: Run `git-kit pr "Feature Name"` to create a plan in `.github/pr-plans/`.
3.  **Approve**: Ask the Maintainer (or AI Agent) to review the plan.
4.  **Implement**: Write code and tests.
5.  **Status**: Use `git-kit status` to track your work.

## Development
1.  Install: `pip install -e .[dev]`
2.  Test: `pytest`
